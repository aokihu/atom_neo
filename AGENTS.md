# 代码编写规范约束
- 遵循"Less code, more power"原则,代码要精简干练
- 避免相同或者类似功能函数或者方法被多次创建，尽可能的复用已有的方法与函数
- 先思考后编写,禁止盲目编写
- 对于目录结构复杂的目录,必须在该目录下编写`README.md`文件,对目录中的文件以及功能做摘要描述,加快代码文件的理解速度
- 须要先更新文档，然后再实施修改（文档驱动开发）

# 文档的规范约束
- 文档统一存放在目录`docs`
- **index.md**是所有文档的索引文档
- 一个`Markdown`文档对应一个`Web`文档，并且内容保持同步
- `Web`文档是给人阅读的,需要将数据可视化以及图表化,禁止直接引用对应的`Markdown`文件
- 文档必须跟随项目开发进度及时更新
